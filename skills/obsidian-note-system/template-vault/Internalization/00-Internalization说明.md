# Internalization

## 定义

Internalization——内化：不是简单理解，而是根据目的、或者任务，把相关最小知识单元组合连接起来，做成产品或工具。

## 这里放什么

- 主题整理
- 任务导向组合
- 结构设计
- 知识之间的连接关系
- 框架化、模块化、工具化过程稿
- 从最小知识单元到方案、产品、工具的中间层

## 这里不放什么

- 还只是原始摘录的素材
- 已经完成并可直接交付的最终成果

## 与另外两层的关系

- `Internalization` 负责把 `Input` 里的最小知识单元重新组织
- 它是从知识单元到成果输出之间的中间层
- 经过这里整理后的内容，可以继续进入 `Output`

## 当前原则

- 围绕目标组织知识
- 围绕任务连接知识
- 强调组合、重组、映射和结构化
- 不停留在“看懂了”
- 要能走向可执行、可产出、可复用
