# Input / Internalization / Output

## 三阶段定义

Input——输入：不是简单收集，而是把学习到的知识化为最小知识单元，作为知识库基础，后可以重复使用。  
Internalization——内化：不是简单理解，而是根据目的、或者任务，把相关最小知识单元组合连接起来，做成产品或工具。  
Output——输出：不是简单表达，而是产出有价值的、可以重复使用的产品。

```text
输入 = Input
内化 = Internalization
输出 = Output
```

```text
目标
 ↓
输入 ←→ 内化 ←→ 输出
 ↑       ↑       ↓
 └──── 反馈与迭代 ────┘
```

## 使用原则

- Input 负责最小知识单元与素材输入
- Internalization 负责连接、组合、重组和任务化
- Output 负责产出可复用结果
- 三者不是线性一次性流程，而是可反馈迭代的循环
- 母版库本身只保留稳定结构、说明页和模板页
