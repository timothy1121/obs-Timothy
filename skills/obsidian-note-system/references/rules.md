# Writing and Governance Rules

## 当前使用规则

- 先判断知识类型，再决定结构
- 不再强迫所有笔记都用同一个四段式
- 一张卡只解决一个核心对象
- 保留 `关联知识`
- 中文简洁
- 机制优先
- 忠实原意
- 不写空泛口号
- 不强行凑字数

## 治理规则

- 正式卡按模板治理
- 素材不强行套模板
- 新增笔记时，先分类，再套模板
- 后续只做增量治理，不再重复全库重排
- 母版库不承载具体知识卡、项目卡、成果卡

## 编辑规则

- 保留标题
- 保留双链和有效 wiki links
- 优先保留原文精度
- 删除模板空话、污染文本、对话残留
- 不要用泛化教材话术替代原有内容
- `.obsidian` 只保留稳定默认配置，不保留个人工作区状态
